# Pulsar 设计思考

> 2026-03-15 brainstorming 记录，保留推导过程和决策依据

---

## 一、Pulsar 的定位演进

### 从书签管理器到知识加工厂

Pulsar 最初是一个链接收藏工具，灵感来自 Poche.app。但收藏本身没有价值——**加工后的知识才有价值**。

核心转变：
- **仓库思维**：存得越多越好 → 链接堆积，从不回顾
- **工厂思维**：每一条信息都有"下一步动作" → 输入必须转化为输出

Pulsar 的真正定位不是"我把链接放哪里"，而是"我如何把读到的东西变成自己的知识"。

### Collector's Fallacy（收藏家谬误）

这是知识工作者最常见的陷阱：

> 收藏了 ≠ 读了 ≠ 理解了 ≠ 能运用

Readwise 里的高亮、Telegram 里转发的链接、Obsidian 里的 Links.md——收集渠道越多，谬误越深。Pulsar 要做的不是增加第 N 个收集渠道，而是在收集和产出之间架一座桥。

## 二、知识工作流的断裂点分析

```
收集（已解决）→ 阅读（在做）→ [断裂] → 思考 → 输出
```

三个断裂点：

**断裂 1：高亮了但不回顾**
- Readwise 存了几千条高亮，但没有触发机制让你重新看到
- 没有 spaced repetition，阅读变成一次性行为
- 解法：间隔重现机制，让高亮主动浮出水面

**断裂 2：回顾了但不加工**
- 看到高亮原文，但没有追问"所以呢？这对我意味着什么？"
- 缺少一个低摩擦的方式把"别人的话"变成"我的理解"
- 解法：processing 状态 + "一句话想法"输入框

**断裂 3：想了但不写下来**
- 灵感转瞬即逝，没有即时捕获的方式
- 想法和阅读材料分散在不同工具里，无法产生化学反应
- 解法：多格式 inbox，让 thought 和 link 混在一起

## 三、学习科学视角

有效学习需要三个要素（来自认知心理学研究）：

### Elaboration（精加工）
用自己的话重述，而非被动高亮。高亮 ≠ 理解。当你写下"这篇文章对我的启发是 X"时，你被迫进行了深度加工——这是记忆从短期转为长期的关键步骤。

**Pulsar 实现**：processing 状态 + "一句话"模板。不要求写长文，一句话就够。关键是强制你加工。

### Spacing（间隔重复）
同样的内容隔一段时间再看，记忆效果远好于一次性阅读。Ebbinghaus 遗忘曲线表明，24 小时后遗忘 70%。

**Pulsar 实现**：高亮间隔重现（0/3/7/30 天）。不需要 Anki 那样复杂，只是把旧内容定期推回视野。

### Interleaving（交叉）
不同主题交替学习比单一主题集中学习效果更好。大脑在切换主题时被迫比较和区分，加深理解。

**Pulsar 实现**：每日队列从不同 category 混合抽取，而非按主题排列。

## 四、Readwise + Pulsar + Obsidian 三角关系

三个工具各有所长，不应互相替代，而应形成闭环：

```
         Readwise
        ↗        ↘
  深度阅读高亮    highlights API
       ↑              ↓
   Obsidian  ←——  Pulsar
   笔记+思考      AI分析+关联
       ↓
     输出
```

### 各工具角色

| 工具 | 角色 | 擅长 | 不擅长 |
|------|------|------|--------|
| **Readwise** | 精读层 | 深度阅读、高亮管理、书籍摘录 | 跨文章分析、AI 加工 |
| **Pulsar** | 速览层 + 加工层 | 广度收集、AI 分析、关联发现 | 深度阅读体验 |
| **Obsidian** | 产出层 | 长文写作、知识连接、永久笔记 | 内容发现、自动化 |

### 数据流

```
输入：
  Telegram → Pulsar（碎片发现，自动）
  Obsidian Links.md → Pulsar（主动收藏，手动触发）
  Readwise → Pulsar（深度阅读 + 高亮，定期同步）

加工：
  Pulsar AI → 分类 / 标签 / 摘要 / 关联
  用户 → processing 状态 + "一句话想法"
  间隔重现 → 旧内容重新浮出

输出：
  Pulsar → Obsidian 笔记（AI 摘要 + 高亮 + 个人想法）
  Pulsar → Weekly Digest（每周阅读回顾）
  Pulsar → 研究报告（主题聚合分析）
```

关键原则：**Readwise 负责阅读，Pulsar 负责加工，Obsidian 负责产出**。三者不重叠。

## 五、MCP 接入 — AI 作为知识的接口

这可能是最被低估的方向。当 Pulsar 作为 MCP tool 接入 Claude：

- "我收藏过哪些关于货币政策的文章？" → Claude 搜索 Pulsar 返回结果
- "帮我总结一下我最近收藏的 AI 相关内容" → Claude 调 API 获取数据后综合分析
- "基于我的阅读记录，推荐我接下来应该读什么" → Claude 分析你的兴趣图谱

**你的收藏变成了 AI 的长期记忆。** Pulsar 不再是一个你需要打开的网页，而是 AI 能主动检索的知识库。

技术上极其简单——Pulsar 已经有 HTTP API，写一个 MCP server 定义 tools 即可。

## 六、技术哲学 — 何时引入复杂度

### 无框架原则

Pulsar 选择原生 HTML/CSS/JS，不用 React/Vue/Vite。这不是技术落后，而是一种产品态度：

- 单文件 SPA，改完刷新即生效
- 无构建步骤，无依赖地狱
- 任何人拿到代码就能理解
- Terminal 暗黑美学本身就是设计语言

### 何时需要 Vite

触发条件（满足任一即考虑）：
- `index.html` 超过 **3000 行**
- 需要引入 **3+ 个 npm 包**
- 需要 JSX/TypeScript/CSS 预处理器
- 多个 JS 文件需要 import/export

当前 index.html ~1700 行，距离阈值还很远。即使做完 V1-V3 所有功能，预计也不会超过 2500 行。

### 何时需要数据库

触发条件：
- 多用户（需要隔离数据）
- 数据量超过 JSON 处理能力（>10 万条）
- 需要复杂查询（JOIN、聚合）

当前几百条链接，JSON 绰绰有余。

### 不过早优化

引用 Knuth：*"Premature optimization is the root of all evil."*

每一层复杂度的引入都应该是**被痛点驱动**的，而非预防性的。当单文件真的痛了再拆，当 JSON 真的慢了再换数据库，当 CDN 引入真的不够了再上 Vite。

## 七、内容衰减与反收藏

### 信息的时间价值

不是所有内容都值得永久保留：

```
时效性内容（ephemeral）：
  - "2024 Q3 Fed 利率决议分析" → 半年后价值 ≈ 0
  - "马斯克又说了什么" → 一周后价值 ≈ 0
  - 大部分新闻、市场评论

常青内容（evergreen）：
  - "价值投资的核心原则" → 十年后仍有价值
  - "如何有效阅读一本书" → 永久有效
  - 方法论、原理、思维模型
```

AI 可以在 analyze.py 阶段自动标记 `ephemeral` / `evergreen`，影响间隔重现的权重和自动 archive 策略。

### 减法的价值

300 篇收藏中真正有价值的可能只有 50 篇。把另外 250 篇 archive 掉，不是损失，是**信噪比的提升**。

每日队列应该优先展示：
1. 常青内容的定期回顾
2. 新收藏的快速分流（值得深读 or archive）
3. processing 状态的待加工项

而非按时间排列的所有内容。

## 八、阅读画像 — 量化自省

你读什么定义了你是谁。阅读画像不是虚荣指标，而是：

- **发现盲区**：大量收藏 AI 内容但缺少 AI safety → 知识结构不完整
- **追踪兴趣迁移**：三个月前关注 DeFi，现在关注 AI → 兴趣在转向
- **评估输入/输出比**：收藏 100 篇但只产出 5 条笔记 → 加工瓶颈

这些洞察比"你这周读了 10 篇文章"有意义得多。

## 附：版本规划逻辑

为什么这样分版本：

**V1（知识加工）**：解决最核心的断裂——从"存了"到"想了到写了"。这是 Pulsar 从工具变成习惯的关键。如果这一步不成功，后面都没有意义。

**V2（数据源 + 搜索）**：扩大输入面（Readwise）+ 找到已有内容的能力（Fuse.js）。让加工流水线有更多原料，让已加工的内容可以被检索。

**V3（智能层）**：向量搜索让"连接"成为可能——自动发现文章之间的关联，AI 帮你做跨文章综合分析。这是从"个人知识库"到"个人知识助手"的跃迁。
